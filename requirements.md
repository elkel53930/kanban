# カンバンアプリ要求仕様書（Ubuntuローカル実行）

## 1. 概要

### 1.1 目的

Ubuntu PC 上でローカルに動作し、Web ブラウザを通じて使用できる個人向けカンバンアプリを開発する。本アプリは、日々のタスク管理を簡潔かつ効率的に行うことを目的とし、最小限の機能と直感的な操作性を提供する。

### 1.2 対象ユーザー

個人利用のみ。利用者は 1 名。

### 1.3 利用環境

* OS：Ubuntu
* ブラウザ：ユーザーが使用中のデフォルトブラウザ
* ネットワーク：ローカル PC のみで完結。LAN 越しアクセスは不要。

---

## 2. システム構成

### 2.1 アーキテクチャ

* PC 内で Web サーバーを起動し、ブラウザ経由でアクセスする構成
* クライアントは HTML/CSS/JavaScript による Web UI
* サーバーは任意の言語・フレームワーク（Python/Flask、Node.js/Express、Rust/Axum など）で構築可

### 2.2 データ保存

* SQLite または JSON ファイル
* ユーザーは特に指定しないため実装側で選定する
* バックアップ機能は不要

---

## 3. 機能要件

### 3.1 カンバンボード

* ボードは **1つ固定**
* カラムは以下の4つに固定

  1. Todo
  2. 今日やる
  3. 実行中
  4. 完了

### 3.2 カード（タスク）

カードが保有する情報は以下とする：

| 項目   | 内容                       |
| ---- | ------------------------ |
| タイトル | 文字列（必須）                  |
| 説明文  | Markdown 形式              |
| 期限   | 日付（任意）                   |
| タグ   | 任意数。テキストラベル。             |
| コメント | 任意数。Markdown 形式。作成日時を保持。 |

#### 操作要件

* ドラッグ＆ドロップでのカード移動
* カードの新規作成、編集、削除
* カード内コメントの追加、削除
* タグの追加・削除

### 3.3 完了カラムの特別動作

* 完了（Done）に移動したカードは、**翌日になると自動的に現在のカンバン上から消える**
* 消えたカードは「履歴ページ」で過去に遡って閲覧できる
* 履歴ページでは以下が可能：

  * 日付別一覧表示
  * カードの内容（タイトル、説明、コメントなど）の閲覧
  * 検索（タイトル/タグなど）

### 3.4 ユーザー管理

* ログイン不要
* 権限管理不要

---

## 4. UI 要件

### 4.1 全体デザイン

* シンプルで軽量な UI
* Trello のような見た目は必要ないが、直感的に使える構成
* モバイル向け表示対応は不要

### 4.2 カンバン画面

* 4 カラムの横並び表示
* 各カードは最小限の情報（タイトル＋期限＋タグ）を表示
* カード詳細はモーダルまたは別画面で表示

### 4.3 履歴ページ

* 日付ごとに完了したタスクを一覧表示
* カレンダーナビゲーションまたは日付リストによる遡り
* カードクリックで詳細閲覧（説明文、コメント含む）

### 4.4 Markdown

* 説明文・コメントは Markdown をレンダリングして表示
* コードブロック、リスト、リンクなど基本的な表現に対応

---

## 5. 非機能要件

### 5.1 パフォーマンス

* 同時アクセス 1 名のため特別な要求なし
* 一般的な PC スペックで十分動作すること

### 5.2 セキュリティ

* 外部公開しないため最低限で良い
* CSRF/XSS 対策は可能な範囲で実施
* 認証機能は設けない

### 5.3 対応プラットフォーム

* Ubuntu のみをターゲットとする

---

## 6. 運用要件

### 6.1 起動方法

* 手動起動
* Ubuntu 起動時の自動起動（systemd）は不要

### 6.2 ログ

* 特にログ機能は不要
* 実装上必要となる最小限のエラーログのみ出力可
